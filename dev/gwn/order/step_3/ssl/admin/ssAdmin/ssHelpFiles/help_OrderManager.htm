Order Manager for StoreFront 5.0

Release Version 1.4
Release Date June 27, 2002

Order Manager permits you to keep track of payments and shipments for your website through your Microsoft Internet Explorer browser. This is accomplished by calling the file ssOrderAdmin.asp from you browser.

General Operation

  1. Once a customer places an order there are two steps that must be performed, collecting payment and filling their order. Order Manager helps you keep track of  both these steps. As soon as an order is placed an entry will appear in the order status summary. You can narrow down the results through the use of the filter.
  2. When you receive payments you can either record the payments indivually through the order detail section or en masse using the quick payment screen.
  3. When you ship the orders you can either record the orders individually through the order detail section or en masse using the quick shipment screen. In conjunction with recording an order as shipped you can optionally send a fulfillment email to your customer.
  4. If you use an external shipping program you can import the shipping information via use of the Shipping Import Screen.

Views

There are three primary views on this page. These are selected by the radio buttons at the top of the filter section.

1. Detail - this view pulls up the detailed order status. You can edit payments and orders in this view. This is the only view you can individually edit the shipment email.

2. Quick Payment - a list of all orders without payments will appear by default. You can enter the payment date and method of payment for all orders at one time. Double clicking the payment date box will automatically fill in today's date.

3. Quick Shipment - a list of all orders that haven't been shipped appear by default. Orders without payments will have a "-" for the payment date. You may double click the shipment date box and today's date will be filled in. You may also fill in the order tracking number if desired in this view. An email will be generated for those orders that are checked.

Filtering

You can narrow down your order summary by entering an order id or customer name. You can select "Do Not Include" to ignore entries in this text box.

Start Date: Only orders placed on or after the value entered in this box will be displayed. If you leave it blank this is ignored.

End Date: Only orders placed on or before the value entered in this box will be displayed. If you leave it blank this is ignored.

You may also filter out orders that have been paid for (or not), orders that have been flagged (or not) and orders that have been shipped (or not.)

Additional Features:

In the detail view are two notes boxes, internal and external. The internal notes can only be viewed on this page. The external notes will appear on the customer order history page and can be used to communicate with the customer.

In the flag this order section there are two back order message boxes, internal and external. The internal message can only be viewed on this page. The back order message will appear on the customer order history page.

The Show Order Summaries button will open up a new window with just the order summaries matching the filter selection.

 - This will open up a new window with just the order detail

Email Notification:

The file ssOrderAdminEmail.txt contains the email template. Within this template are fields marked with <> that are used to fill in the actual values from the database such as customer name, order number, shipment date, etc. It is important that you do not remove the lines that say "DO NOT REMOVE THIS LINE" At the bottom of this file is a list of constants that are available to use to customize your template.

Order Status

Both the date payment received and date order shipped fields can be automatically populated with the current date by double clicking the field name. Alternatively, you can click the calendar icon to open a calendar window to select the date. Finally, you can just type in the date. You can set the payment type to the payment method the customer selected by double clicking the field name.

You can edit the text of the optional email message by clicking the "Edit Email Text" button. It will be populated with the actual email message based upon the order tracking number and shipping method selected. The email message will only be sent if you select the "Send Email" checkbox.

Changing Shipping Information

To change the shipping method name or shipping charge, simply double click the shipping method name. Placing a cursor over the name will change it to a hand. You can then enter the new information in the boxes provided. You must hit the update button to save the changes.

Importing External Shipping Information

  1. To import shipping information following the link from the Order Administration page to the Import Shipping Information Page.
  2. Browse for the shipping data file to import
  3. You can optionally deselect emails you do not wish to send
  4. You may select Automatically update unrecorded payments to automatically mark orders which are being shipped but not previous marked as paid using today's date.
  5. Press Import

The data file must be in the following format:

OrderID,  Shipping Method, Tracking Number,  Shipping Date

The fields may be either separated by commas or tab with each entry being separated by a carriage return. Shipping Date is optional; if it is left out it will be assumed to be today's date.
