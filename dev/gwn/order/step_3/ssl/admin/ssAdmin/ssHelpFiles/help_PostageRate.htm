User's Guide for Sandshot Software's Postage Rate Component

Configuration
Customization

The Postage Rate Component obtains real time rate information from FedEx, Freight Quote, United Parcel Service, and the United States Postal Service. It is highly configurable to estimate your customers' order shipping costs. Each individual shipping method can be configured to adjust the returned rate based on a variety of features.

 As an additional feature, insurance rates can be obtained based upon the declared value. However, the insurance rates from UPS and U.S.P.S. themselves are NOT real time. The insurance rates presented are based upon Domestic Insurance rates in effect as of January 28, 2001. The insurance rate from FedEx is calculated real time.

The file CarrierAdmin.asp allows you to edit your shipping methods. Please note that for the real-time rates to work, the shipCode field must NOT be altered. You may change the name however.

Configuration

Each shipping method must be identified to a carrier. Each shipping method contains a variety of features which permit you to customize each method to meet your needs.

Shipping Methods

  • checkbox - this identifies which methods are to be updated or deleted
  • Shipping Methods
    • Name - this is the name that appears in your site
    • Code - this is a unique code as required by the carrier
    • Carrier - the carrier this method is unique to. You can administer your carrier list via ssshippingMethodsAdmin.asp
    • Enabled - check if you want it to be available for your customers
  • Shipping Charges
    • Min - this sets the minimum charge for the methods' rate and is used if the carrier returns a rate less than this amount
    • Mult. - this is used to multiply the rate returned by the carrier. It will also apply to the minimum charge if used
    • Pkg Fee - this is a per package fee assessed on each box being shipped. You may have more than one box if the "super package" exceeds identified limits
    • Ship Fee - this is a per shipment fee and is only applied once, no matter how many boxes are created
  • Free Shipping
    • Amount - this identifies the order amount which qualifies for this method to have free shipping.
    • Max Wt - if the order weight exceeds this amount, free shipping will not be applied
  • Order Restrictions
    • Min Wt - the minimum order weight before this shipping method appears as a choice. It is primarily used for freight
    • Max Wt - the maximum an order can weigh and still show this shipping method as a choice. It is also used to break down a "super package"
  • Package Restrictions
    • Pref Wt - the weight you prefer to have the "super package" broken down by. This value is useful because some carriers start charging excessively for heavier packages. Example. If an order weighs 70 lbs, it may be cheaper to send it as two 35 lb packages than one 70 lb package. If you set the preferred weight to 35 lbs and the max weight to 70 lbs, as long as no individual item weighs more than 35 lbs the calculator will estimate the order based on two 35 lb packages.
    • Max Girth - if you have larger, light items it is possible to exceed the carriers' max package dimensions while still meeting their absolute weight limit. This value permits you to handle this situation.
  • Default - select the shipping method you wish to be selected in the dropdown

You can manually arrange the display order of the shipping methods by selecting the desired shipping method and moving it up or down the list with the and buttons.

Carriers

  • Carrier Name - self explanatory
  • Username - some carriers require a username to access their site
  • Password - some carriers require a password to access their site

Rate calculation. The order item are automatically loaded to identify the individual items in the order and their respective sizes. If there are multiple items in the order they are combined into a "super package" which simply creates a package by summing the package heights and using the maximum length and width. The weights are themselves summed. The "super package" is then compared against the maximum weight and maximum dimensions permitted for the shipping method. If these maximums are exceeded, the "super package" is divided into smaller packages. This division will be based upon the maximum item weight and the preferred weight. If no individual item weighs more than the preferred weight, the package will be divided by the preferred weight. A package cannot be created which is smaller than the maximum item weight nor the largest individual dimension. The rate is then determined by submitting the resulting package to the carrier for rating. The carrier will return a rate which is the basis for the shipping method calculation as follows:

Shipping Charge = Number of Packages x ((Multiple x Rate) + per Package Fee) + Shipment Fee
where the rate is returned is the rate returned by the carrier or the minimum charge for that shipping method, whichever is greater.

FedEx

There are no user configurable elements.

United Parcel Service

UPS has a number of  potential customizations.

COD: set pblnCODInd = True if package is to be sent COD

Hazardous Material: set pblnHazMat = True if the package contains hazardous material

Additional Handling Charge: set pblnAdditionalHandlingInd = True if the package requires additional handling

Call Tag Service: set pbytCallTagARSInd = one of the following

0  if No Call Tag (default)
1  if using Basic Call Tag service
2  if using Electronic Call Tag service

Saturday Delivery: set pblnSatDeliveryInd = true if using Saturday delivery

Saturday Pickup: set pblnSatPickupInd = true if using Saturday pickup

Print Return Label: set pblnReturnLabelInd = True if using this feature

Residential or Commercial address:

Set pblnResidentialInd = False if shipping to a commercial address
Set pblnResidentialInd = True if shipping to a residential address (default)

Delivery Confirmation Service: set pbytDCISInd = one of the following

0 for None (default)
1 for Basic 
2 for Signature Required
3 for Alternate Return Address 
4 for All Available Information

Verbal Confirmation of Delivery: set pblnVerbalConfirmationInd = True if you wish Verbal Confirmation of Delivery.

First Ship Notification: set pbytSNDestinationInd1 = one of the following

0 for None  (default)
1 for Domestic 
2 for International

Second Ship Notification: set pbytSNDestinationInd2 = one of the following

0 for None  (default)
1 for Domestic 
2 for International

PackagingType: set pstrstrPackagingType = one of the following

00 for Shipper Supplied Packaging (default)
01 for UPS Letter Envelope 
03 for UPS Tube 
21 for UPS Express Box
24 for International UPS 25KG Box 
25 for International UPS 10KG Box

United States Postal Service

There are no user configurable elements.
